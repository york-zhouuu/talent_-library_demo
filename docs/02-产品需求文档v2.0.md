## 1. 背景与目标

### 1.1 背景（沿用）

**基建枢纽定位**：人才库是所有服务（传统B端、Agent服务）都会用到的核心基建。

**当前痛点**：
- 人才库几乎停摆，无人使用
- 大量猎头维护私有库（Excel、纸笔），成熟猎头手上有200-300+份简历
- 组织维度 sourcing 不透明、效率低
- Agent 无法介入（好人才分布在私有库）

### 1.2 核心升级：从"搜索"到"对话"

| 旧版本 (v1.x) | 新版本 (v2.0) |
|---------------|---------------|
| 关键词搜索 | 对话式搜索（告诉 Agent 要什么人） |
| 列表筛选 | Agent 推荐 + 理由 + 对话优化 |
| 人工筛选 | AI 初筛 + 人工确认 |
| 静态字段 | AI 理解的人才画像 |

### 2.3 与 OpenClaw 的集成设计

#### 2.3.1 Skill 设计

人才库 Agent 需要暴露以下 Skill 给 OpenClaw：

```yaml
# talent_search skill
skill: talent_search
description: "搜索人才库，找到符合条件的候选人"
parameters:
  - name: query
    type: string
    description: "自然语言描述人才需求"
  - name: source
    type: enum
    options: [private, public, all]
    default: all
  - name: limit
    type: integer
    default: 10

# talent_detail skill
skill: talent_detail
description: "获取候选人详细信息"
parameters:
  - name: talent_id
    type: string

# talent_recommend skill
skill: talent_recommend
description: "基于候选人推荐相似人才"
parameters:
  - name: talent_id
    type: string
  - name: reason
    type: string
    description: "为什么推荐相似人才"

# talent_import skill
skill: talent_import
description: "批量导入简历到私有库"
parameters:
  - name: files
    type: array
    description: "简历文件列表"

# talent_update skill
skill: talent_update
description: "更新候选人信息"
parameters:
  - name: talent_id
  - name: updates
    type: object
```

### 3.2 核心功能模块

#### 3.2.1 数据导入（沿用 + 增强）

| 功能 | 说明 | OpenClaw 集成 |
|------|------|---------------|
| 一键批量导入 | PDF/Word 简历批量解析 | 暴露 `talent_import` skill |
| AI 手机自动入库 | 电话后自动生成候选人 | 通过 OpenClaw 触发 |
| AI Meeting 自动入库 | 会议后自动入库 | 通过 OpenClaw 触发 |

**迭代点**：
- 导入过程可以调用 Agent 自动分类/标签
- 支持对话式导入确认："这份简历是 Java 工程师，对吗？"

#### 3.2.2 对话式搜索（核心新功能）

| 旧版 | 新版 (Agent 驱动) |
|------|------------------|
| 输入关键词 | 自然语言描述需求 |
| 列表结果 | Agent 推荐 + 理由 |
| 手动筛选 | 对话式优化 |
| "还要会 Python" | "帮我排除 35 岁以上的" |

**对话示例**：

```
顾问: 我有个客户要招 Go 工程师，5 年以上
Agent: 明白，5 年以上 Go 工程师。找到 35 位候选人，推荐前 5：
  [1] 张三 | 7年 | 字节跳动 | Go + Python | 期望 60-80W
      推荐理由：在字节主导过 Go 微服务架构...
  [2] 李四 | 5年 | 快手 | Go | 期望 50-60W
      ...

顾问: 客户base北京，薪资控制在 50W 以内
Agent: 好的，base 北京 + 50W 以内，更新为 12 位候选人...
  [1] 李四 | 5年 | 百度 | Go | 45W
      ...

顾问: 还要有海外工作背景
Agent: 符合条件的有 3 位...
  [1] 王五 | 6年 | Meta(前) | Go | 40W
      推荐理由：有 2 年硅谷工作经历...
```

#### 3.2.3 智能推荐

| 功能 | 说明 |
|------|------|
| 职位匹配 | 输入职位描述，Agent 自动推荐 |
| 相似推荐 | 选择候选人，Agent 推荐相似的 |
| 智能排序 | 根据顾问偏好 + 历史反馈排序 |

#### 3.2.4 候选人沟通（Agent 驱动）

| 功能 | 说明 |
|------|------|
| 初步接触 | Agent 自动打招呼，收集意向 |
| 需求确认 | Agent 确认候选人的求职意向 |
| 跟进提醒 | Agent 提醒顾问跟进候选人 |

#### 3.2.5 人才库管理

| 模块 | 说明 |
|------|------|
| 私有库 | 加密存储，仅本人可访问 |
| 公有库 | 共享池，按权限开放 |
| AI 画像 | Agent 理解的候选人摘要（脱敏） |

## 4. 数据模型

### 4.1 候选人表

```typescript
interface Candidate {
  // 基础信息
  id: string;
  name: string;
  phone: string;           // 加密存储
  email?: string;          // 加密存储
  
  // 职业信息
  current_company?: string;
  current_title?: string;
  work_years?: number;
  location?: string;
  
  // 期望
  expected_salary?: string;
  expected_location?: string;
  
  // 技能
  skills: string[];
  
  // AI 画像（非原始数据，供 Agent 使用）
  ai_summary?: string;     // Agent 理解的摘要
  ai_evaluation?: string;  // Agent 的评价
  
  // 库归属
  library_type: 'private' | 'public';
  owner_id?: string;       // 私有库时
  
  // 状态
  status: 'active' | 'passive' | 'unreachable';
  tags: string[];
  
  // 原始简历
  resume_url?: string;
  
  // 元数据
  created_at: Date;
  updated_at: Date;
}
```

### 4.2 私有库信任机制

```typescript
// 加密方案：用户密钥加密
interface PrivateLibrary {
  id: string;
  owner_id: string;
  
  // 加密密钥由用户端管理，服务端不存储
  // 查询时：用户传入密钥 → 解密 → 返回
  
  // 仅存储脱敏数据供公有搜索
  public_summary: {
    skills: string[];
    work_years_range: string;
    location: string;
    // 无联系方式、无具体公司
  };
}
```

## 5. OpenClaw Skill 定义

### 5.1 人才搜索 Skill

```yaml
name: talent_search
description: "在人才库中搜索符合条件的候选人"
parameters:
  type: object
  properties:
    query:
      type: string
      description: "自然语言描述的人才需求，如：5年以上的Java工程师"
    library:
      type: string
      enum: [private, public, all]
      default: all
    limit:
      type: integer
      default: 10
      maximum: 50
  required: [query]

examples:
  - "帮我找 5 年以上的 Go 工程师"
  - "北京地区的 Java 候选人"
  - "薪资 50 万以内的产品经理"
```

```yaml
      type: array

        type: string
```

## 7. 风险与应对

### 7.1 核心风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 私有库数据泄露 | 信任崩塌 | 端到端加密 + 审计日志 |
| Agent 推荐不准确 | 用户流失 | 人工确认 + 反馈优化 |
| 数据导入成本高 | 顾问不使用 | 一键极致简化 |

### 7.2 OpenClaw 集成风险

| 风险 | 影响 | 应对 |
|------|------|------|
| Agent 响应慢 | 体验差 | 搜索结果缓存 + 分页 |
| 多轮对话上下文丢失 | 搜索中断 | Memory 持久化 |
| 私有库查询权限 | 数据泄露 | 用户密钥隔离 |
